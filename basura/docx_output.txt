Para 13: 3. Metodología y Mejores Prácticas Aplicadas
Para 14: 3.1 Arquitectura de firmware (top-down)
Para 15: Siguiendo las mejores prácticas de sistemas embebidos industriales (Betasolutions, FreeCodeCamp), el firmware se estructura como una Máquina de Estados Finita (FSM) con estados bien definidos. Esto permite trazabilidad de comportamiento, manejo robusto de errores y facilidad de prueba.
Para 16: INIT: configuración de periféricos, validación de hardware al encendido.
Para 17: READING: adquisición periódica de temperatura con promedio móvil de 8 muestras (ADC 12-bit).
Para 18: COOLING: TEMP_INT > TEMP_EXT — compuertas abiertas + ventilador ON.
Para 19: IDLE: TEMP_INT ≤ TEMP_EXT — compuertas cerradas + ventilador OFF.
Para 20: ERROR: lectura inválida, sensor desconectado o fuera de rango → estado seguro (todo OFF).
Para 21: 3.2 Conversión NTC (Ecuación Beta)
Para 22: El sensor NTC TT05-10KC8-1S-T105-1500 (R₀=10kΩ @ 25°C, B=3435K [corregido desde 3950K per datasheet TEWA/TME]) requiere un divisor de voltaje Vout = Vcc × R_ref / (R_ntc + R_ref) para convertir resistencia en voltaje ADC. La temperatura se calcula con la ecuación Beta (precisión ±0.5°C en rango operativo):
Para 23: T(K) = 1 / [ (1/T₀) + (1/B) × ln(R_ntc / R₀) ]
Para 24: Se aplica promedio móvil de 8 lecturas ADC y se filtran valores fuera del rango físico del sensor (−40°C a 105°C, Tmax per part number T105). Un ADC de 12-bit produce error < 0.05°C según AllAboutCircuits, suficiente para la aplicación.
Para 25: 3.3 Etapa de potencia para ventilador 48V
Para 26: El ventilador MR1238E48B-FSR opera a 48V DC. Los GPIOs del RP2350 son de 3.3V y no pueden alimentarlo directamente. Se implementa una etapa con MOSFET NMOS (ej. IRL520N) con resistencia de gate (100Ω), diodo flyback (1N4007) y pull-down en gate para estado seguro en ausencia de señal. El GPIO4 (VENT) controla el gate a través del resistor.
Para 27: 3.4 PCB mixed-signal: separación analógica/digital
Para 28: Siguiendo las guías de Analog Devices y PCBWay para diseño mixed-signal, el layout del PCB separa físicamente la sección de potencia (TB6612FNG + MOSFET de ventilador) de la sección de señal analógica (divisores NTC + entradas ADC). Se usa plano de GND unificado con punto estrella para evitar lazos de corriente que degraden las lecturas ADC.
Para 29: 3.5 Histéresis y estado seguro
Para 30: Se implementa histéresis de ±1°C alrededor del setpoint de temperatura para evitar oscilación rápida del actuador (chattering). El watchdog timer del RP2350 está habilitado con timeout de 8 segundos; al dispararse, el sistema entra en estado seguro (todo OFF) antes del reset. Durante el travel del actuador FIT0803 (~3s bloqueante), el WDT se alimenta cada 500ms desde hbridge._safe_sleep_ms() vía callback set_wdt_feed(), evitando reset accidental.

Para 151: 5. Decisiones Técnicas Clave y Justificación
Para 152: ► Lenguaje: MicroPython
Para 153: MicroPython sobre C/C++ permite iteración rápida en el plazo de 4 días, acceso directo a periféricos RP2350 (machine.ADC, machine.Pin, machine.WDT) y legibilidad de código para la sesión en vivo. El ciclo de control a 1Hz no requiere el rendimiento de C.
Para 154: ► Conversión NTC: ecuación Beta
Para 155: La ecuación Beta (B=3435K, corregido desde 3950K per datasheet TEWA/TME) ofrece precisión de ±0.5°C en el rango 0–85°C, suficiente para control de temperatura de gabinete. La ecuación Steinhart-Hart completa requiere calibración de 3 puntos que no está disponible en el entorno de evaluación.
Para 156: ► Resistencia de referencia NTC: 10kΩ
Para 157: Igual a R₀ del sensor, esto maximiza la sensibilidad de la lectura ADC en el rango centrado en 25°C, reduciendo el error de cuantización en la zona de operación más común.
Para 158: ► MOSFET NMOS para ventilador 48V
Para 159: El IRL520N (Vgs(th)=2V, Id=10A, Vds=100V) es accionable directamente por el GPIO 3.3V del RP2350, soporta la corriente del ventilador y tiene encapsulado TO-220 soldable en PCB.
Para 160: ► Histéresis ±1°C
Para 161: Evita el chattering (conmutación rápida) del actuador FIT0803 al cruzar el setpoint de temperatura, aumentando la vida útil del mecanismo y reduciendo ruido eléctrico en el sistema.
Para 162: ► Plano GND unificado en PCB
Para 163: Analog Devices y las guías KiCad recomiendan GND unificado (en lugar de tierra partida) para señales mixtas de baja frecuencia, con punto estrella para separar retornos de potencia de los analógicos y evitar corrientes de lazo que degraden las lecturas ADC.

Para 177: 8. Mejoras y Optimizaciones Implementadas en la Simulación Wokwi (C++/Arduino)
Para 178: Durante el desarrollo de la simulación en Wokwi (sketch.ino), se implementaron mejoras sustanciales para facilitar la evaluación técnica y cumplir rigurosamente con los requerimientos del PDF original:
Para 179: 1. Parpadeo Dinámico del Ventilador: Para simular la máxima capacidad de extracción solicitada en los requerimientos, el LED indicador del ventilador parpadea a una frecuencia de 150ms cuando la temperatura interna es mayor o igual a 60°C, manteniendo un encendido fijo a temperaturas menores.
Para 180: 2. Comandos vía Monitor Serial: Se implementó una interfaz de comandos por terminal para facilitar la demostración de la simulación sin la necesidad de manipular físicamente los componentes iterativamente:
Para 181:   - SET:XX -> Fija manualmente la temperatura de activación (Setpoint), anulando la lectura del DIP switch.
Para 182:   - TEMP:XX -> Inyecta una temperatura interna simulada de forma instantánea.
Para 183:   - EXT:XX -> Fija de forma manual la temperatura exterior simulada.
Para 184: 3. Lógica FSM No Bloqueante: Todo el código de la máquina de estados y las transiciones (como el tiempo de viaje de la compuerta de ~3 segundos) fue rediseñado usando millis() para garantizar que el loop principal nunca se bloquee, manteniendo la frecuencia de control a 1 Hz estricto.
Para 185: 4. Remoción de inyecciones automáticas: Se eliminaron los retardos forzados (delays) que inyectaban temperatura al mover el DIP switch, delegando esta función a los comandos Serial para una simulación térmica más pura y controlable.
